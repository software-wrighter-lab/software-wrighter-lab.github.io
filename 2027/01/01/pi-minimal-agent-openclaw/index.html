    Four tools. That’s it. Read, Write, Edit, Bash.

    Pi is the minimal coding agent created by Mario Zechner, and it’s the engine behind OpenClaw—the agent framework that’s becoming the future of agentic coding.

    The Philosophy: Minimal Core, Maximum Capability

    Most agents try to do everything. Pi does almost nothing—and that’s the point.

    Agent Approach Tools Philosophy
    Traditional Dozens Built-in everything
    Pi 4 Extensions for the rest

    Pi’s four core tools:

    • Read: Read files
    • Write: Write files
    • Edit: Edit files
    • Bash: Run commands

    Everything else is an extension.

    What is OpenClaw?

    OpenClaw is an agent framework built on Pi’s components. It connects an agent to a communication channel of your choice and just runs code.

    The key insight: LLMs are really good at writing and running code, so embrace this.

    Architecture

    ┌─────────────────────────────────────┐
│         Communication Layer         │  Telegram, CLI, etc.
├─────────────────────────────────────┤
│            OpenClaw                 │  Agent Framework
├─────────────────────────────────────┤
│              Pi Core                │  4 tools + extensions
├─────────────────────────────────────┤
│         Custom AI SDK               │  Multi-provider support
└─────────────────────────────────────┘

    Key Features

    Feature Description
    Tree sessions Sessions stored as trees, enabling branching and navigation
    Multi-provider Switch models within single sessions
    Hot reloading Develop extensions without restarting
    TUI rendering Spinners, progress bars, data tables

    The Extension System

    Extensions can:

    • Register tools
    • Create TUI components
    • Function as skills
    • Persist state into sessions

    Armin Ronacher’s extensions (in mitsuhiko/agent-stuff):

    • answer: Response formatting
    • todos: Task tracking
    • review: Code review
    • control: Flow control
    • files: File operations

    MCP Support via mcporter

    Pi doesn’t have built-in MCP support. Instead, OpenClaw uses mcporter, which exposes MCP calls via:

    • CLI interface
    • TypeScript bindings

    Why Minimal Wins

    The fascination with minimal agents: software that builds more software.

    1. Tiny system prompt - Pi has the shortest system prompt of any agent
    2. Composable - Build what you need
    3. Debuggable - Less magic, more transparency
    4. Fast - No bloat

    Key Takeaways

    1. Less is more. Four tools plus extensions beats dozens of built-in features.

    2. LLMs write code. Embrace it instead of fighting it.

    3. Extensions over features. Let agents build their own tools.

    4. The future is minimal. OpenClaw’s growth suggests this approach wins.
